== ワークブックの保護

=== 概要

ワークブックの保護機能により、以下のユーザー操作を保護できます (パスワード付きまたはパスワードなし)。

* ウィンドウの保護: 個別の子ウィンドウに各ワークブックを表示する古いバージョンの MS Excel で、ワークブックのMDI 子ウィンドウのサイズ変更する機能
* ワークブック構造: ワークシートの追加、名前変更、削除、非表示、順序変更を行う機能。

.注:
[NOTE]
====
{ExcelEngineName} のオブジェクト モデルから保護が強制されることはありません。これらの保護設定を履行し、対応する操作の実行をユーザーに許可または制限することは、このオブジェクト モデルを表示する UI の責任です。
====

=== 構成チャート

以下の表は、ワークブックの保護機能の要素を簡単に説明し、それらを操作するプロパティとメソッドにマップします。

[options="header", cols="a,a,a"]
|====
|作業|詳細|プロパティ / メソッド

|ワークブックの保護設定
|
ifdef::wpf[]
`Protect` メソッドのオーバーロードを使用して、ワークブックをパスワード (`string` または `SecureString` を使用して指定) で保護、またはパスワードなしで保護することができます。このメソッドのその他の引数は、前述で説明した各保護設定の指定に使用されます。
endif::wpf[]
ifdef::xamarin[]
ワークブックを保護するには、`Protect` メソッドを使用できます。このメソッドの引数は、前述で説明した各保護設定の指定に使用されます。
endif::xamarin[]
| link:{ExcelEngineApiLink}infragistics.documents.excel.workbook~protect.html[Protect]

|ワークブックの保護解除
|
ifdef::wpf[]
`Unprotect` メソッドのオーバーロードを使用して、パスワード付きまたはパスワードなしで設定したワークブックの保護を解除できます。 

`Unprotect` メソッドに対して指定されたパスワードがワークブックに設定されたパスワードと一致しない場合、例外がスローされます。 

.注: 
[NOTE] 
==== 
パスワードで保護されたワークブックであっても、パスワード引数なしで `Unprotect` オーバーロード メソッドを使用して保護を解除できます。 
====
endif::wpf[]
ifdef::xamarin[]
`Unprotect` メソッドを使用して、パスワード付きまたはパスワードなしで設定したワークブックの保護を解除できます。
endif::xamarin[]
| link:{ExcelEngineApiLink}infragistics.documents.excel.workbook~unprotect.html[Unprotect]

|ワークブックの保護の有無のチェック
|この読み取り専用プロパティは、ワークブックに `Protect` メソッドpick:[wpf="のオーバーロード"]を使用して設定された保護がある場合、True を返します。
| link:{ExcelEngineApiLink}infragistics.documents.excel.workbook~isprotected.html[IsProtected]

|ワークブックのパスワード保護のチェック
|この読み取り専用プロパティは、ワークブックがパスワードで保護されている場合に True を返します。
| link:{ExcelEngineApiLink}infragistics.documents.excel.workbook~hasprotectionpassword.html[HasProtectionPassword]

|ワークブックの保護の取得
|この読み取り専用プロパティは、保護の各設定を個別に取得するためにプロパティを含む link:{ExcelEngineApiLink}infragistics.documents.excel.workbookprotection.html[WorkbookProtection] 型のオブジェクトを返します。
| link:{ExcelEngineApiLink}infragistics.documents.excel.workbook~protection.html[Protection]

|====
