== Adding a Trend Line

=== Overview

The trend line displays in the  _{SparklineName}_   control as another layer on top of the Sparkline layer. To display a trend line, use the link:{SparklineLink}.{SparklineName}~trendlinetype.html[TrendLineType] property.

The trend lines are calculated according to the algorithm specified by the link:{SparklineLink}.{SparklineName}~trendlinetype.html[TrendLineType] property using the values of the data that the chart is bound to. The supported trend lines are listed and explained in the {SparklineName} Overview topic.

Trend line types can be displayed only one at a time. (No multiple trend lines can be displayed simultaneously.)

By default, the trend line is not displayed.

=== Code Example

The following code demonstrates setting of the link:{SparklineLink}.{SparklineName}~trendlinetype.html[TrendLineType] property to QuinticFit.
ifdef::sl,wpf,win-phone,win-universal,xamarin[]
*In XAML:*

[source,xaml]
----
<igSparkline:{SparklineName} TrendLineType="QuinticFit"/>
----
*In C#:*

[source,csharp]
----
this.{SparklineName}1.TrendLineType = SparklineTrendType.QuinticFit;
----

ifdef::sl,wpf,win-phone,win-universal[] 
*In Visual Basic:*

[source,vb]
----
Me.{SparklineName}1.TrendLineType = SparklineTrendType.QuinticFit
----
endif::sl,wpf,win-phone,win-universal[] 
endif::sl,wpf,win-phone,win-universal,xamarin[] 
ifdef::win-forms[] 
*In C#:*

[source,csharp]
----
this.{SparklineName}1.TrendLineType = TrendLineType.QuinticFit;
----


*In Visual Basic:*

[source,vb]
----
Me.{SparklineName}1.TrendLineType = TrendLineType.QuinticFit
----
endif::win-forms[] 

=== Configuring the Trend Line

The Trend Line can be configured in the following aspects:

* Trend line type
* Trend line period
* Trend line color
* Trend line thickness

For details on how to configure the trend lines, refer to the link:sparkline-configuring-the-trend-line.html[Configuring the Trend Line] topic.
