== Geographic Imagery

[[_Ref320488759]]

=== Overview

Geographic imagery is a detailed representation of the world from a top view perspective. It can consist of an aerial-satellite map or road maps in a multi-scale imagery tiles structure. The _{GeoMapName}_ control can display geographic imagery in order to provide end-users with rich and interactive world maps and geographic context for geo-spatial data.

[[_Ref320700551]]

[[_Ref320489029]]
== Map Background Content

[[_Ref320489035]]

=== Description

The _{GeoMapName}_ control’s pick:[wpf="link:{GeoMapLink}.{GeoMapName}~backgroundcontent.html[BackgroundContent] property is used to display"]pick:[win-forms="link:{GeoMapLink}.{GeoMapName}~setimagery.html[SetImagery] method is used to set the BackgroundContent to any of"] all supported types of geographic imagery sources. For each geographic imagery source, there is an imagery class used for rendering corresponding geographic imagery tiles.

The following table summarizes imagery classes provided by the _{GeoMapName}_ control.

[options="header", cols="a,a"]
|====
|Imagery Class|Description

| link:{GeoMapLink}.geographicmapimagery_members.html[GeographicMapImagery]
|Represents the base control for all imagery classes that display all types of supported geographic imagery tiles. pick:[wpf="This class can be extended for the purpose of implementing support for geographic imagery tiles from other geographic imagery sources such as Map Quest mapping service. For more information on this feature, refer to the link:geographicmap-creating-custom-geographic-imagery-from-map-quest.html[Creating Custom Geographic Imagery from Map Quest] topic."]

| link:{GeoMapLink}.openstreetmapimagery_members.html[OpenStreetMapImagery]
|Represents the multi-scale imagery control for displaying geographic imagery tiles from the Open Street Maps service.

| link:{GeoMapLink}.azuremapsimagery_members.html[AzureMapsImagery]
|Represents the multi-scale imagery control for displaying geographic imagery tiles from the Azure Maps service.

| link:{GeoMapLink}.bingmapsmapimagery_members.html[BingMapsMapImagery]
|Represents the multi-scale imagery control for displaying geographic imagery tiles from the Bing Maps service.

|====

By default, the pick:[wpf="link:{GeoMapLink}.xamgeographicmap~backgroundcontent.html[BackgroundContent] property"]pick:[win-forms="BackGroundContent"] is set to link:{GeoMapLink}.openstreetmapimagery_members.html[OpenStreetMapImagery] object and the _{GeoMapName}_ control displays geographic imagery tiles from the Open Street Maps service. In order to display different types of geographic imagery tiles, the _{GeoMapName}_ control must be re-configured.

The following is a list of topics that provide information on how to display different types of geographic imagery tiles:

* link:geographicmap-displaying-geographic-imagery-from-open-street-maps.html[Displaying Geographic Imagery from Open Street Maps]
* link:geographicmap-displaying-geographic-imagery-from-azure-maps.html[Displaying Geographic Imagery from Azure Maps]
* link:geographicmap-displaying-geographic-imagery-from-bing-maps.html[Displaying Geographic Imagery from Bing Maps]
ifdef::wpf[]
* link:geographicmap-creating-custom-geographic-imagery-from-map-quest.html[Creating Custom Geographic Imagery from Map Quest]
endif::wpf[]

ifdef::wpf[]
In addition, the link:{GeoMapLink}.xamgeographicmap~backgroundcontent.html[BackgroundContent] property can be set to any object that inherits the class. However, only objects that inherit the link:{GeoMapLink}.geographicmapimagery_members.html[GeographicMapImagery] class will allow panning and zooming of the map background content.
endif::wpf[]

In the _{GeoMapName}_ control, map background content is always rendered behind all geographic series. In other words, geographic imagery tiles are always rendered first and any geographic series in the _{GeoMapName}_ control’s link:{DataChartLinkBase}.seriesviewer~series.html[Series] property is rendered on top of the geographic imagery tiles. This is especially important when displaying multiple geographic series in the same plot area of the _{GeoMapName}_ control because geographic imagery tiles can quickly get buried in the map view. For more information on this, refer to the following topics:

* link:geographicmap-hiding-geographic-imagery-in-map-background-content.html[Hiding Geographic Imagery in Map Background Content]
* link:geographicmap-displaying-multiple-geographic-series.html[Displaying Multiple Geographic Series]

=== Code

This code example explicitly sets pick:[wpf="link:{GeoMapLink}.xamgeographicmap~backgroundcontent.html[BackgroundContent]"]pick:[win-forms"BackgroundContent"] of the _{GeoMapName}_ control to the link:{GeoMapLink}.openstreetmapimagery_members.html[OpenStreetMapImagery] object which provides geographic imagery tile from the Open Street Maps.

ifdef::wpf[]
*In XAML:*

[source,xaml]
----
<ig:XamGeographicMap x:Name="GeoMap">
    <ig:XamGeographicMap.BackgroundContent>
        <ig:OpenStreetMapImagery />
    </ig:XamGeographicMap.BackgroundContent>
</ig:XamGeographicMap>
----
endif::wpf[]

*In Visual Basic:*

[source,vb]
----
Dim GeoMap As New {GeoMapName}()
ifdef::wpf[]
Me.GeoMap.BackgroundContent = New OpenStreetMapImagery()
endif::wpf[]
ifdef::win-forms[]
Me.GeoMap.SetImagery(New OpenStreetMapImagery())
endif::win-forms[]
----

*In C#:*

[source,csharp]
----
var geoMap = new {GeoMapName}();
ifdef::wpf[]
this.geoMap.BackgroundContent = new OpenStreetMapImagery();
endif::wpf[]
ifdef::win-forms[]
this.geoMap.SetImagery(new OpenStreetMapImagery());
endif::win-forms[]
----
