== Overview

In order to distinguish activities' owner the {SchedulerName} control supports *resources*. A resource may be a person or some other entity and is represented by the link:{SchedulerCoreLink}.scheduleresource.html[ScheduleResource] class. One resource may own one or more activities. Also all activities associated with this resource are going to be rendered using the resource's color scheme. Each resource needs to have a unique id.

== Main Properties

The following table lists the `Resource`{empty}'s key properties and their purpose:

[options="header", cols="a,a"]
|=======
|Property|Purpose

|link:{SchedulerCoreLink}.scheduleresource~id.html[Id]
|The resource's ID must be unique amongst all resources.

|link:{SchedulerCoreLink}.scheduleresource~displayname.html[DisplayName]
|The display name identifies a resource in the application's user interface.

|link:{SchedulerCoreLink}.scheduleresource~colorscheme.html[ColorScheme]
|The color scheme is used to highlight the activities associated with this resource.
